## 🚶 页面 4: 行进模式主屏 ⭐ **最复杂**

### 业务场景

用户正在徒步中，需要:

- 实时查看当前位置
- 查看距离下一个关键点的距离
- 查看已走距离/剩余距离/配速
- 查看队伍成员位置 (如有)
- 快速触发 SOS
- 暂停/继续/结束记录

### 必须展示的信息

```
地图区域 (占主要屏幕):
  ☐ 用户当前位置 (高亮标记)
  ☐ 路线轨迹 (已走路径 + 未走路径, 颜色区分)
  ☐ 关键点标记 (下一个高亮)
  ☐ 队伍成员位置 (实时更新)
  ☐ 缩放/平移控件

顶部卡片 (悬浮):
  ☐ 下一个关键点名称
  ☐ 距离下一个关键点 (如 "↑ 800m")
  ☐ 预计到达时间 (如 "约 15 分钟")

底部数据面板:
  ☐ 已走距离 / 总距离
  ☐ 已用时间
  ☐ 当前配速 (如 "12'30\" /km")
  ☐ 累计爬升
  ☐ 暂停/继续按钮
  ☐ 结束按钮
  ☐ SOS 按钮 (红色, 醒目)

侧边栏 (可展开):
  ☐ 队伍成员列表 (头像/昵称/距离我的距离/状态)
  ☐ 关键点列表 (已过/未过)
```

### 交互说明

```
☐ 地图跟随模式 (自动居中用户位置)
☐ 手动拖动地图 → 暂停跟随 (显示"返回跟随"按钮)
☐ 点击队伍成员 → 地图移动到成员位置
☐ 点击关键点标记 → 显示关键点详情
☐ 上滑底部面板 → 展开详细数据
☐ 暂停 → 数据停止更新, 按钮变为"继续"
☐ 结束 → 二次确认 → 进入复盘页
☐ SOS → 单独设计 (见页面 6)
```

### UX 动效说明 ⭐ **重要**

```
必须的动效:
  ☐ 用户位置标记 → 呼吸动画 (表示实时)
  ☐ 接近关键点 → 关键点放大 + 弹跳提醒
  ☐ 队伍成员移动 → 位置平滑过渡 (不突变)
  ☐ 数据更新 → 数字滚动动画
  ☐ 暂停/继续 → 按钮变形动画
  ☐ SOS 按钮 → 轻微震动提示 (可点击)

可选的动效:
  ☐ 到达关键点 → 完成打勾动画 + 震动反馈
  ☐ 轨迹绘制 → 路径渐现动画 (首次进入)
```

### 设计参考

```
参考:
  - Strava 记录界面 (数据展示)
  - 高德导航界面 (地图 + 数据卡片)
  - Keep 跑步界面 (数据面板)
  - Apple Watch 体能训练 (简洁数据)

设计要点:
  - 信息层次: 地图最大, 数据次要, SOS 突出
  - 对比度高 (户外阳光下可见)
  - 按钮大 (易于点击, 戴手套也能操作)
  - 数据实时性 (避免延迟感)
  - 省电考虑 (深色模式 for OLED)
```
